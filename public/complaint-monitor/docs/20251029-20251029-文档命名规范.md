## 📝 命名规则

### 基本格式

```
YYYYMMDD-YYYYMMDD-文件名.md
│        │        │
│        │        └─ 描述性中文文件名
│        └────────── 最后更新日期
└─────────────────── 创建日期
```

### 示例

| 场景 | 文件名 | 说明 |
|-----|--------|------|
| 新建文档 | `20251029-20251029-第一阶段完成报告.md` | 创建和更新日期相同 |
| 更新文档 | `20251029-20251030-第一阶段完成报告.md` | 更新日期后移 |
| 技术方案 | `20251028-20251029-投诉技术方案.md` | 记录迭代历史 |
| 开发日志 | `20251029-20251029-开发日志-Day1.md` | 按日记录 |

## 📋 命名规范详细说明

### 1. 日期格式

**格式**: `YYYYMMDD`（8位数字）

| 部分 | 说明 | 示例 |
|-----|------|------|
| YYYY | 4位年份 | 2025 |
| MM | 2位月份（01-12） | 10 |
| DD | 2位日期（01-31） | 29 |

**完整示例**: `20251029` = 2025年10月29日

### 2. 分隔符

使用 **连字符 (-)** 分隔日期和文件名

```
20251029-20251029-文件名.md
        ↑        ↑
     分隔符   分隔符
```

### 3. 文件名部分

**要求**:
- ✅ 使用中文
- ✅ 简洁明了
- ✅ 描述准确
- ❌ 避免特殊字符
- ❌ 避免空格（用连字符代替）

**好的文件名**:
- ✅ `第一阶段完成报告`
- ✅ `投诉监控技术方案`
- ✅ `数据库设计文档`
- ✅ `API接口说明`

**不好的文件名**:
- ❌ `report1` (使用英文)
- ❌ `第 一 阶 段` (包含空格)
- ❌ `方案@v1.0` (包含特殊字符)
- ❌ `文档` (不够描述性)

### 4. 文件扩展名

统一使用 `.md` (Markdown格式)

## 🔄 更新规则

### 更新文件时的操作

1. **重命名文件**
   ```bash
   # 旧文件名
   20251029-20251029-第一阶段完成报告.md
   
   # 新文件名（更新日期后移）
   20251029-20251030-第一阶段完成报告.md
   ```

2. **更新文档头部信息**
   ```markdown
   **创建时间**: 2025-10-29  
   **更新时间**: 2025-10-30  ← 更新这里
   ```

3. **添加变更记录**
   ```markdown
   ## 📝 变更历史
   
   ### 2025-10-30
   - 更新了配置说明
   - 添加了监控指标
   
   ### 2025-10-29
   - 初始版本
   ```

## ⚠️ 注意事项

1. **日期格式必须严格遵守**: `YYYYMMDD`（8位数字）
2. **创建日期不可修改**: 始终保持文档创建时的日期
3. **更新日期需同步**: 更新内容时必须同步更新文件名
4. **中文文件名**: 使用UTF-8编码，确保跨平台兼容
5. **避免重名**: 同一天创建的同类型文档需要有区分度

## 🔍 验证工具

### Shell脚本验证

```bash
#!/bin/bash
# 验证文件名是否符合规范

FILE="$1"
PATTERN="^[0-9]{8}-[0-9]{8}-.+\.md$"

if [[ $FILE =~ $PATTERN ]]; then
    echo "✅ 文件名符合规范: $FILE"
else
    echo "❌ 文件名不符合规范: $FILE"
    echo "正确格式: YYYYMMDD-YYYYMMDD-文件名.md"
    exit 1
fi
```

### 使用示例

```bash
./validate_filename.sh "20251029-20251029-第一阶段完成报告.md"
# 输出: ✅ 文件名符合规范

./validate_filename.sh "report.md"
# 输出: ❌ 文件名不符合规范
```
